# Challenges in Representation Learning: Facial Expression Recognition Challenge
ეს რეპოზიტორია შექმნილია Kaggle-ის შეჯიბრის **"Challenges in Representation Learning: Facial Expression Recognition Challenge"** ფარგლებში. პროექტის მიზანია ადამიანის სახის მიმიკების კლასიფიკაცია 7 ძირითად ემოციად (*Angry, Disgust, Fear, Happy, Sad, Surprise, Neutral*).
პროექტი სრულად არის ინტეგრირებული **Weights & Biases (WandB)** პლატფორმასთან ექსპერიმენტების თრეკინგის, ჰიპერპარამეტრების მონიტორინგისა და მოდელების შენახვისთვის. ძირითად სამუშაო 
სივრცეს წარმოადგენდა **Google Colab**

## რეპოზიტორიის სტრუქტურა

└── README.md -> რეპოზიტორიის თითოეული ფაილის აღწერა და დეტალური გარჩევა

## Data Preprocessing
მოდელის წვრთნის დაწყებამდე, უმნიშვნელოვანესი ეტაპი დაეთმო მონაცემთა მომზადებასა და გაწმენდას, რაც აღწერილია `data_preprocessing.ipynb` ნოუთბუქში.
Kaggle პლატფორმიდან გადმოვიტანე შეჯიბრის შესაბამისი დატასეტი და დავიწყე მისი გაცობით. დავადგინე რა ზომისა და ფორმის მონაცემებთან გვქონდა საქმე. ასევე, ძალიან 
მნიშვნელოვანია მონაცემებში ემოციების შესაბამისი სურათების განაწილება. ამის თვალსაჩინოებისთვის ავაგე გრაფიკი

<img width="528" height="297" alt="image" src="https://github.com/user-attachments/assets/2f799a7d-5b17-426a-afa4-f0a8ced94cba" />

გავიგე ზუსტი რაოდენობებიც. გამოიკვეთა კლასების დისბალანსი. ეს შეიცავდა რისკს, რომ მოდელს დაეიგნორებინა მცირე კლასები. ამიტომ, გამოვიყენე 
წონების განაწილების სისტემა, რომელიც შემდგომ დანაკარგის ფუნქციას გადაეცემა და აიძულებს მოდელს, მაქსიმალურად თანაბრად ისწავლოს ყველა ემოცია. 

ამის შემდეგ უშუალოდ მონაცემთა დამუშავებაზე გადავედი. 
 `pixels` სვეტში არსებული ტექსტური მონაცემები გადაყვანილ იქნა `float32` მასივებად, მიიღო $48 \times 48$ ფორმატი და გაიყო `255.0`-ზე, რათა მნიშვნელობები მოქცეულიყო $[0, 1]$ შუალედში. 
 `train_test_split`- ის გამოყენებით მონაცემები დაიყო პროპორციით 85/15. `stratify=y_all` პარამეტრის დახმარებით, 7-ვე ემოციის კლასობრივი განაწილება საწვრთნელ და სავალიდაციო სეტებში შენარჩუნდა იდენტური.
`matplotlib`- ის საშუალებით მოხდა შემთხვევითი სურათების და მათი ლეიბლების ვიზუალური შემოწმება, რამაც დაადასტურა მონაცემთა სტრუქტურის სისწორე.
